**Out of scope by design.** DSL semantics, runtime engine, claw-code-go, sandbox engine, cloud deployment plumbing.

Severity model:
- **P0** — exploitable security flaw, data loss, crash on common path, auth bypass.
- **P1** — UX broken on a real path, perf bloquante (>1 s lag on a frequent action), absence of user-visible failure for a load-bearing fetch, measurable goroutine/memory leak, defence-in-depth defect that materially weakens an existing protection.
- **P2** — meaningful debt with maintenance cost, refactor justified, a11y gap that doesn't break a parcours.
- **P3** — polish, naming, lint.

## 2. Production-readiness

### 2.1 Security
- **HttpOnly cookie defeated by JSON body** — F-S1 (P1, shipped). Auth response echoed the JWT both as HttpOnly cookie and as `access_token` field, making the protection cosmetic for browser sessions.
- **`OpenExternal` Wails binding accepts arbitrary URL schemes** — F-S2 (P1, shipped). Any JS in the WebView (XSS, malicious artifact path bound to a handler) could invoke `file://`, `ms-msdt:`, custom URI handlers.
- **`?t=<jwt>` query-param token rejected on `/api/ws`** — F-X1 (P1, shipped). Browser cookie path masks the bug; non-browser WS clients (CLI, SDK) get 401 even with a valid token.
- **`safePath` reads `s.cfg.WorkDir` without `stateMu`** — F-S3 (P1, deferred). Project hot-swap can race a concurrent file save → containment check uses the old prefix while the new project root is being installed. Correctness/data-integrity issue, not a direct security bypass; see §7.
- **Preview proxy reaches loopback/private addresses in local mode** — F-S4 (P2, deferred). Authenticated SSRF in multi-user local deployment; not a P1 because the typical local mode is single-user.

No P0s: there is no XSS sink, no eval, no markdown injection, uploads are sanitized, cross-store `?store=` is symlink-safe via `EvalSymlinks` + prefix containment, WS Origin allowlist is correct, mime sniffing on disk matches `Allowed Upload MIMEs`, secrets in the desktop keychain never reflect to JS.

### 2.2 Robustness / WS
- **WS log subscriptions silently dropped on Resume/Cancel** — F-R1 (P1, shipped). `reconnectToken` change wiped `logSubscriberCountRef`/`logsRequestedRef`; consumer panels stayed mounted; new ws.onopen never re-subscribed.
- **Event-history fetch failure invisible to the user** — F-R2 (P1, shipped). The store rolls back its `historyFetchedForRun` marker but the UI rendered an empty Events tab with no toast — looked like data loss.
- **Slow WS client pins streamEventsLocal goroutine up to ~10 s per write stall** — F-R3 (P1, deferred). Bounded by `writeWait`, but under many frozen clients accumulates. See §7 for the recommended drop-on-full pattern.

### 2.3 Performance
- **`blobToBase64` O(N²) string growth** — F-P1 (P1, shipped). `binary += String.fromCharCode(...)` copies an ever-growing string on each chunk; multi-MB artifact downloads freeze the UI for seconds.
- **`listArtifacts` re-fired on every WS event** — F-P2 (P1, shipped). `useEffect` depended on the whole `exec` object; parent rebuilds `executions` on each event so `exec` identity churns; tens of /artifacts requests in flight in parallel during an active node.
- **Single bundle 3.0 MB / 882 kB gzip** — P2 cluster. Vite itself warns; no `React.lazy` / `Suspense` in the tree. Code-split is deferred — non-trivial because the Wails AssetServer embed needs all chunks resolvable as static assets. See §8.

### 2.4 Observability
- No structured front-end telemetry, but `events.jsonl` server-side suffices for the model the project explicitly committed to in `docs/adr/`. WSStatusDot exists; we did not add a permanent disconnect banner — the existing reconnect logic is sound and a banner would risk false positives during normal Resume/Cancel-driven reconnect.

### 2.5 Build & deployment
- `task editor:build` + `task desktop:build` (Wails, tag `desktop,webkit2_41`) both green locally where headers are present. Wails AssetServer rejects WS upgrades by design; SPA dials loopback directly. Current local desktop builds omit `?t=` because `GetSessionToken` returns an empty string; future hosted-auth desktop builds should append a token only when the binding returns one.
- Bundle size warning persists post-fixes — explicit P2 in §8.

## 5. Cross-layer contracts

- **WS protocol symmetry**: client cases `snapshot|event|event_batch|log_chunk|log_terminated|terminated|error|ack` ↔ server `wsType*` constants. Mutating frames `subscribe|unsubscribe|cancel|answer|subscribe_logs|unsubscribe_logs` validated server-side via `sendError("unknown_type", ...)`. No drift.
- **`?t=` query-param token**: was rejected on `/api/ws` (no trailing slash) → fixed (F-X1, shipped). Still gated to WS routes only — does not leak onto regular HTTP paths.
- **`?store=` cross-store override**: `isSafeStoreParam` is duplicated client-side in both `api/runs.ts` and `useRunWebSocket.ts`. Defence-in-depth, intentional. Server-side `resolveCrossStore` calls `EvalSymlinks` on both input and root before prefix check. No symlink escape.
- **Wails AssetServer**: cannot accept WS upgrades (501 by design); SPA dials loopback directly. Local desktop currently omits any token query because `GetSessionToken` returns `""`; tokenized WS dials apply only to future hosted-auth desktop flows that return a token. `asset_proxy.go` rewrites Origin on proxied `/api/*` HTTP requests to satisfy the CORS allowlist but does not forward auth cookies (desktop servers run `DisableAuth=true`).

## 7. Findings — detailed

Each finding lists severity, file:line, impact, status (✅ shipped in this branch / ⏭ deferred), and the commit hash if shipped.

### F-S1 — `access_token` echoed in auth response body for browser sessions [P1] ✅ `8db6a313`
- **File**: `pkg/server/auth_routes.go:140,498,519` (3 sites). 
- **Impact**: HttpOnly cookie protection undermined — any future XSS can read the JWT via `fetch('/api/auth/login').then(r=>r.json()).then(d=>d.access_token)`.
- **Fix**: `isBrowserClient(r)` heuristic (Sec-Fetch-* / Origin headers); body field only populated for non-browser callers (CLI/SDK).

### F-S2 — `OpenExternal` Wails binding has no scheme validation [P1] ✅ `a677da72`
- **File**: `cmd/iterion-desktop/bindings.go:191`.
- **Impact**: WebView JS (XSS or malicious artifact bound to handler) can invoke `file://`, `javascript:`, `ms-msdt:`, custom URI handlers via `wruntime.BrowserOpenURL`.
- **Fix**: parse + reject schemes outside http/https before forwarding to Wails.

### F-X1 — `?t=<jwt>` rejected on `/api/ws` (no trailing slash) [P1] ✅ `9ecb9f1e`
- **File**: `pkg/server/middleware.go:110`.
- **Impact**: Non-browser WS clients (CLI, SDK) cannot authenticate to the file-event hub even with a valid token. Browser-only clients work because of cookies. With test coverage in `middleware_test.go`.
- **Fix**: accept both exact `/api/ws` and `/api/ws/*` prefix.

### F-S3 — `safePath` reads `s.cfg.WorkDir` without `stateMu` [P1] ⏭ deferred
- **File**: `pkg/server/server.go:1078` (and any caller during `handleSwitchProject` window).
- **Impact**: Race during project hot-swap → a concurrent `POST /api/files/save` can pass the old containment check, then the write may land under the new (different) WorkDir or vice versa. Correctness/data-integrity, not direct security.
- **Why deferred**: needs a careful audit of every `s.cfg.WorkDir` reader in `pkg/server/`; safer as its own PR with a test that exercises switch-mid-save. Tracked as a follow-up.

### F-R1 — Log subscriptions silently dropped on Resume/Cancel [P1] ✅ `4222d132`
- **File**: `editor/src/hooks/useRunWebSocket.ts:111,317`.
- **Impact**: After Resume/Cancel the live log stream stops with no UI cue; user must navigate away and back to recover.
- **Fix**: track `prevRunIdRef`, only reset subscriber state on a real runId change, leave it intact across `reconnectToken` bumps.

### F-R2 — Event-history fetch failure invisible to user [P1] ✅ `3fd0b34f`
- **File**: `editor/src/components/Runs/RunView.tsx:266`, `editor/src/store/run.ts:391`.
- **Impact**: 5xx or transient network error on `/events` rendered Events tab and scrubber empty with no explanation — looked like data loss.
- **Fix**: pipe rejection through `useUIStore.getState().addToast(...)`. The store still rolls back its marker so a retry can succeed.

### F-R3 — Slow WS client pins broker subscription up to ~10 s per write stall [P1] ⏭ deferred
- **File**: `pkg/server/runs_ws.go:863`.
- **Impact**: A frozen client that holds TCP but doesn't drain WS frames keeps `streamEventsLocal` blocked while `writeWait=10 s` ticks; many frozen clients accumulate.
- **Why deferred**: bounded behaviour; introducing drop-on-full requires choosing between "send `error` then close" vs "close silently" and tuning the threshold — best done with a stress-test harness rather than landed blind.

### F-P1 — `blobToBase64` O(N²) on multi-MB artifacts [P1] ✅ `da5093f5`
- **File**: `editor/src/api/runs.ts:494`.
- **Impact**: Desktop user downloading a 10 MB artifact sees the UI freeze for seconds while the string-concat loop progresses.
- **Fix**: push chunks into an array, `join('')` once at end.

### F-P2 — `listArtifacts` re-fired on every WS event for the focused node [P1] ✅ `8984c7f2`
- **File**: `editor/src/components/Runs/NodeDetailPanel.tsx:196`.
- **Impact**: During an active node the browser's HTTP connection pool saturates with redundant `/artifacts` fetches, slowing log chunks and snapshot refreshes.
- **Fix**: depend on stable scalars (`runId`, `ir_node_id`, `execution_id`); route an `AbortController` into `listArtifacts` (added `signal` option to the API helper).

### F-P3 — 3 MB single bundle, no code-splitting [P2 cluster] ⏭ deferred
- **File**: `editor/vite.config.ts`, all routes in `editor/src/App.tsx`.
- **Impact**: First paint penalty on slow networks; embedded SPA in the desktop binary is correspondingly large.
- **Why deferred**: introducing `React.lazy` per route needs a Suspense boundary strategy + verification that the Wails AssetServer correctly serves dynamic chunks (the embed is generated by `pkg/server/static/`). Worth a dedicated PR.

### F-U1 — French strings in NodeDetailPanel tool-payload UI [P1] ✅ `4a72ff6c`
- **File**: `editor/src/components/Runs/NodeDetailPanel.tsx:1036,1047,1048`.
- **Impact**: Non-French operators face unlabeled affordances on the main UX for reading large tool outputs.
- **Fix**: translated to English (`expand`/`collapse`, `loading…`, `load more`).

### F-U2 — A11y under-coverage cluster [P2 cluster] ⏭ deferred
- **Files**: 41/91 `onClick`-bearing files lack `aria-*`; two `<div onClick>` backdrops in `ShortcutsHelp.tsx:35`, `SettingsDrawer.tsx:59`.
- **Why deferred**: Radix Dialog handles Escape and traps focus; backdrops are styled overlay click-to-dismiss. Systematic IconButton labeling is a follow-up sprint, not a P1.

### F-X2 — Hardcoded `/api/` in LogLinesView log copy/download [P1] ✅ `047fb0e4`
- **File**: `editor/src/components/Runs/LogLinesView.tsx:391,419`.
- **Impact**: In cloud or any deployment with `VITE_API_URL` ≠ `/api`, the two buttons 404 silently.
- **Fix**: route through `apiURL(...)` (already used everywhere else).

### F-C1 — Auth has no rate limiting on `/api/auth/login|register|refresh` [P2] ⏭ deferred
- **File**: `pkg/server/auth_routes.go:886` (comment acknowledges the gap).
- **Why deferred**: rate limiting strategy is a project-wide design choice (in-process token bucket vs Redis-backed for the cloud path); not appropriate for a frontend-review PR.

### F-C2 — `RevealInFinder` on Windows doesn't scope the path [P2] ⏭ deferred
- **File**: `cmd/iterion-desktop/bindings.go:198`.
- **Why deferred**: `exec.Command("explorer", "/select,"+path)` doesn't shell-out (no injection), but the function accepts any path; a malicious run artifact path could direct the file manager to arbitrary files. Lower-impact than F-S2. Worth a containment check in a follow-up.

### F-R4 — `loadEventHistoryIfMissing` micro-race [P2] ⏭ deferred (downgraded from reviewer's P1)
- **File**: `editor/src/store/run.ts:391`.
- **Why downgraded**: concurrent calls produce at most one redundant fetch; `applyEventsBatch` dedupes by seq, so result is correct. Pure efficiency, not a user-visible bug.
