display_name: Greenfield (new project / feature)
description: Bias every persona toward establishing clean foundations for net-new work
---
This run is a GREENFIELD effort — net-new project or a feature with
little existing code to fit into.

- Analyst: frame the problem broadly; the open questions are about
  product intent, not legacy constraints.
- PM: it's fine to propose foundational stories (scaffolding, the
  first vertical slice) alongside feature stories.
- Architect: favour clean, obvious foundations and clear extension
  points over cleverness. Record the foundational decisions as ADRs —
  they set the project's direction.
- Dev: establish conventions deliberately; a new project's first
  patterns become its house style.

Prefer simplicity and a working first slice over completeness.
