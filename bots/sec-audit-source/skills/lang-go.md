# lang-go — Go scanners

Activated by the `run_go_scanners` branch when `detect_tech` reports
`go` in `tech.langs`. Project layout signals: `go.mod` at workspace
root and any `*.go` files.

## Primary scanner — semgrep

```bash
semgrep \
  --config=p/golang \
  --config=p/owasp-top-ten \
  --json \
  --output={{vars.scan_dir}}/go-semgrep.json \
  --error \
  --metrics=off \
  --quiet \
  --exclude='vendor' \
  --exclude='**/*_test.go' \
  --exclude='**/testdata' \
  {{vars.workspace_dir}}
```

## Secondary scanner — gosec

```bash
gosec \
  -fmt=json \
  -out={{vars.scan_dir}}/gosec.json \
  -exclude-dir=vendor \
  -exclude-dir=testdata \
  -exclude-generated \
  -severity=low \
  -confidence=low \
  -quiet \
  ./...
```

Run from `{{vars.workspace_dir}}` (gosec needs the module root).

gosec rule → `finding_type` mapping (high-signal rules only;
triage may downgrade or merge):

| gosec rule | finding_type | severity floor |
|---|---|---|
| G101 (hardcoded credentials) | `secrets` | high |
| G102 (binding all interfaces) | `config` | medium |
| G103 (unsafe block) | `other` | low (often necessary) |
| G104 (errors not checked) | `other` | low (style, mostly) |
| G201/G202 (SQL string format) | `injection` | high |
| G203 (HTML template not escaped) | `xss` | high |
| G204 (exec with subprocess) | `injection` | high |
| G301-G307 (file permissions) | `config` | medium |
| G401/G402 (weak crypto, bad TLS config) | `crypto` | high |
| G404 (weak RNG for security) | `crypto` | high |
| G501-G505 (broken/disabled crypto) | `crypto` | high |
| G601 (loop var capture, pre-Go 1.22) | `other` | low |

## Framework-specific threat hints

### net/http
- Handlers that read `r.URL.Query()` or `r.PostFormValue()` then
  pass straight to `template.HTML(...)` → `xss`.
- `http.Redirect(w, r, target, ...)` where `target` is request-
  derived → `redirect`.
- Server with no read/write timeouts → `config` (low; DoS surface).

### Gin
- Routes registered without an auth middleware (no `r.Use(auth)` or
  per-route `Group.Use(auth)`).
- `c.Param("id")` flowing directly into a DB query → `idor`.
- `c.Redirect(http.StatusFound, urlFromQuery)` → `redirect`.

### Echo
- `e.Static("/", "public")` exposing source dirs → `path-trav`.
- `c.QueryParam("url")` passed to `http.Get` → `ssrf`.

### Fiber
- `c.SendFile(c.Query("file"))` without path normalisation →
  `path-trav`.

### Chi
- `r.Mount` without a verifying middleware on the mounted router →
  `authz`.

## Module-level concerns

- `replace` directives in `go.mod` pointing to local paths or git
  forks for crypto-adjacent modules (`crypto/`, `golang.org/x/crypto`,
  jwt libs) → `other` with `dependency-replace-trust` label.
- `init()` functions in third-party modules that touch `os.Setenv`,
  `os.Exec`, network — flag in triage as `other`/`high` (real
  malware audit is sec-audit-deps's job; here we surface the *own
  repo's* init() side effects).

## Excluded patterns

Don't flag:
- `*_test.go`
- `testdata/**`
- `vendor/**` (audited separately by sec-audit-deps)
- Generated code (lines containing `// Code generated by`)
